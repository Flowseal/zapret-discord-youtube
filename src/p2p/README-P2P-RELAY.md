P2P Relay Module for Zapret
Описание
Модуль децентрализованного P2P-релея с максимальной защитой добровольцев (выходных узлов) для обхода геоблокировок.
Ключевые особенности
🔒 Защита узлов

    End-to-end шифрование: узел видит только зашифрованный трафик, не контент
    Фильтрация рискованного контента: встроенные механизмы защиты
    Короткие сессии: автоматическая ротация каждые 10 минут
    Маскировка под HTTPS: трафик выглядит как обычное соединение с легитимным доменом

🌐 Децентрализация

    Нет центральных серверов — пользователи добровольно становятся узлами
    Автоматический выбор лучшего узла по пингу и уровню доверия
    Устойчивость к блокировкам — сеть живёт пока есть участники

⚡ Производительность

    Минимальные задержки за счёт прямого P2P соединения
    Автоматический fallback на стандартный режим zapret
    Совместимость с существующими стратегиями DPI-обхода

Архитектура
Client --(Encrypted)--> Relay Node --(Encrypted)--> Target Site
     |                           |                              |
  Zapret DPI                 Sees only                       Sees relay
  Bypass                     encrypted                       IP, not
                           traffic                          client IP
Безопасность
Для пользователей

    Весь трафик шифруется end-to-end
    DPI-маскировка от провайдера
    Автоматическая смена узлов при блокировках

Для добровольцев (узлов)
Важно: Узел видит только зашифрованный трафик, не содержимое запросов.

    Вы не видите контент — только зашифрованные байты
    Фильтры защищают — блокировка известных рискованных категорий
    Короткие сессии — сложно привязать к конкретному действию
    Маскировка — трафик выглядит как обычный HTTPS к легитимным доменам

Структура модуля
src/p2p/relay.h
Заголовочный файл с определениями структур и функций:

    struct p2p_peer — информация о пир-узле
    struct p2p_config — конфигурация релея
    Функции инициализации и обработки пакетов

src/p2p/relay.c
Реализация модуля:

    p2p_relay_init() — инициализация модуля
    p2p_relay_handle_packet() — обработка пакета
    p2p_verify_peer() — проверка безопасности узла

Интеграция в Zapret
1. Инициализация в main.c
Пример кода:
struct p2p_config cfg = {
    .enabled = 1,
    .secure_mode = 1,
    .max_session_time = 10,
    .fake_domain = "cdn.cloudflare.com"
};
p2p_relay_init(&cfg);
2. Обработка пакетов в packet_handler.c
Пример кода:
// Перед основной DPI обработкой
if (p2p_relay_handle_packet(pkt, peer) == 0) {
    return DROP; // дроп если узел не доверенный
}
3. Конфигурация
Добавить в блок конфигурации:
p2p_relay_enabled = 1
p2p_relay_secure_mode = 1
p2p_relay_max_session_time = 10
p2p_relay_fake_domain = cdn.cloudflare.com
Использование
Быстрый старт
Запуск с P2P релеем:
./zapret --p2p-relay=on --dpi-desync=fake
Становление узлом (доброволец)
Включить режим узла:
./zapret --p2p-relay=node --p2p-relay-secure=1
Важно: Узел работает только в безопасном режиме — вы не видите контент трафика.
FAQ
Q: Безопасно ли быть узлом?
A: Да, если включён secure_mode=1. Вы видите только зашифрованный трафик, не содержимое.
Q: Что если через мой узел сделают что-то незаконное?
A: Трафик зашифрован end-to-end, вы не можете контролировать контент. Рекомендуется:

    Использовать secure_mode=1
    Включить фильтрацию категорий
    Использовать короткие сессии (10 мин)

Q: Можно ли использовать со своими стратегиями?
A: Да, P2P-релей работает поверх существующих стратегий zapret.
Q: Какую нагрузку создаёт узел?
A: Зависит от количества клиентов. Обычно 1-10 Mbps достаточно.
